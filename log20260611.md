# 프로젝트 로그: 인공지능 시인 (AI POET) 구축기
**날짜:** 2026년 6월 11일  
**작성자:** artistjini

## 1. 프로젝트 개요
Google의 최신 LLM인 Gemini 모델과 LangChain 프레임워크를 활용하여, 사용자가 제시한 주제에 맞춰 아름다운 시를 창작해주는 웹 애플리케이션 '인공지능 시인'을 개발하고 배포하는 프로젝트를 수행했다.

## 2. 학습 및 기술 스택 상세
프로젝트를 진행하며 스스로 학습하고 적용한 핵심 개념들은 다음과 같다.

### **LangChain (랭체인)**
- **개념:** LLM(대규모 언어 모델)을 효율적으로 활용하기 위한 라이브러리 프레임워크.
- **언제/왜 사용하는가:** 단순히 모델에 질문을 던지는 것을 넘어, 프롬프트 관리, 모델 체이닝, 외부 데이터 연결 등을 구조화하기 위해 사용했다. 이를 통해 코드의 유지보수성과 확장성을 높였다.

### **LLM 체인 (LLM Chain)**
- **개념:** 프롬프트 템플릿(Prompt Template), 언어 모델(LLM), 출력 파서(Output Parser)를 하나의 흐름으로 연결한 구조.
- **언제/왜 사용하는가:** 사용자의 입력을 정해진 포맷으로 모델에 전달하고, 모델의 응답을 원하는 형식으로 가공하여 사용자에게 보여주기 위해 설계했다. `prompt | llm | output_parser`와 같은 선언적인 구문을 활용했다.

### **Gemini API & Google GenAI**
- **개념:** Google의 최첨단 멀티모달 모델인 Gemini를 활용할 수 있게 해주는 API 인터페이스.
- **언제/왜 사용하는가:** 한국어 시 창작 능력이 뛰어나고 응답 속도가 빠른 `gemini-1.5-flash-latest` 모델을 엔진으로 채택하여 서비스의 품질을 높였다.

### **Streamlit & Streamlit Community Cloud**
- **개념:** Python 코드만으로 빠르고 아름다운 웹 인터페이스를 구축할 수 있게 해주는 라이브러리와 배포 클라우드 서비스.
- **언제/왜 사용하는가:** 별도의 HTML/JS/CSS 지식 없이도 시인 앱의 UI를 구현하고, 전 세계 어디서나 접속 가능한 URL로 배포하기 위해 선택했다.

### **환경 구성: Miniconda**
- **개념:** Anaconda의 경량 버전으로, 필요한 패키지만 골라 설치할 수 있는 파이썬 가상 환경 관리 도구.
- **언제/왜 사용하는가:** 프로젝트별로 독립적인 라이브러리 의존성을 관리하여 개발 환경의 충돌을 방지하고 깔끔한 개발 환경을 유지하기 위해 사용했다.

## 3. 개발 및 배포 단계별 기록

### **단계 1: 로컬 개발 환경 구축 및 코드 작성**
1.  **가상 환경 설정:** Miniconda를 사용하여 프로젝트 전용 환경을 구축했다.
2.  **의존성 관리:** `requirements.txt` 파일을 만들어 `langchain`, `google-genai`, `streamlit` 등 필요한 라이브러리를 정리했다.
3.  **핵심 로직 구현 (`main.py`):**
    - `init_chat_model`을 통해 Gemini 모델을 불러왔다.
    - 사용자의 입력("주제")을 받아 시 작성을 요청하는 프롬프트 템플릿을 설계했다.
    - Streamlit UI를 통해 제목, 입력 필드, 버튼, 로딩 스피너(spinner)를 배치했다.

### **단계 2: Git을 활용한 버전 관리 및 GitHub 연동**
1.  **브랜치 관리:** 기존의 `master` 브랜치를 현대적인 표준인 `main` 브랜치로 변경하여 형상 관리를 시작했다.
2.  **전역 설정 최적화:** 향후 모든 프로젝트에서도 `main` 브랜치를 기본으로 사용하도록 Git 전역 설정을 변경했다.
3.  **GitHub CLI 활용:** `gh` CLI를 사용하여 원격 저장소(`https://github.com/artistjini/ai-poet.git`)와 동기화하고 코드를 푸시했다.

### **단계 3: Streamlit Community Cloud를 통한 배포 및 트러블슈팅**
배포 과정에서 마주한 문제들을 스스로 분석하고 해결하며 실전 역량을 키웠다.

-   **이슈 1: 의존성 설치 오류 (Error installing requirements)**
    - **원인 분석:** `requirements.txt` 파일에 로컬 PC의 특정 경로 정보가 포함되어 클라우드 환경에서 읽지 못하는 것을 발견했다.
    - **해결:** 파일 형식을 표준에 맞게 재작성하고 불필요한 문구를 제거하여 배포를 성공시켰다.
-   **이슈 2: API 인증 오류 (ChatGoogleGenerativeAIError)**
    - **원인 분석:** 클라우드 환경에서는 로컬의 `.env` 파일을 사용할 수 없다는 점을 인지했다.
    - **해결:** Streamlit 대시보드의 **Secrets** 설정을 통해 `GOOGLE_API_KEY`를 안전하게 등록하여 인증 문제를 해결했다.
-   **이슈 3: 할당량 초과 (429 RESOURCE_EXHAUSTED)**
    - **원인 분석:** Gemini API 무료 티어의 요청 제한(Quota)을 확인했다.
    - **해결:** 로그 분석을 통해 현재 사용량과 재시도 가능 시간을 확인했으며, 향후 안정적인 서비스를 위해 할당량 관리의 중요성을 학습했다.
